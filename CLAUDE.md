Техническое задание: Онлайн-сервис создания фотоколлажей
1. Общее описание
Необходимо разработать браузерный SPA-сервис для создания фотоколлажей. Пользователь загружает фотографии, выбирает макет, настраивает внешний вид и скачивает результат в виде изображения (PNG с прозрачным фоном или JPG с фоном).
Стек: React + TypeScript + Vite. Никаких серверных зависимостей — всё работает на клиенте.

4. Компоненты и их поведение
4.1 Рабочая область (Canvas)

Квадратный Canvas по умолчанию (1200×1200 px), размер задаётся пользователем
Отображает текущий коллаж в реальном времени
Каждая ячейка макета — прямоугольная зона с фотографией внутри (object-fit: cover)
Ячейки разделены зазором (gap = ширина рамки)
Клик по ячейке — выделение; можно переназначить фото
Поддержка перетаскивания фото из нижней панели в ячейку Canvas
Фото в ячейке можно двигать и масштабировать (drag внутри ячейки, колесо мыши)

4.2 Зона загрузки фотографий (Dropzone)

Расположена внизу экрана, высота ~140px
Внутри: иконка + текст «Перетащите фото сюда или нажмите для выбора»
При загрузке появляются миниатюры фото (64×64px) в горизонтальной строке
Поддерживаемые форматы: JPG, PNG, WEBP, HEIC (через конвертацию)
Множественный выбор файлов через <input multiple>
Миниатюру можно перетащить в ячейку рабочей области
Кнопка «×» на миниатюре удаляет фото из библиотеки

4.3 Панель настроек (левый сайдбар)
4.3.1 Выбор макета

Горизонтальный скролл с превью-иконками макетов (SVG-иконки, как на скриншоте)
Минимальный набор макетов (16 штук):

1 фото (целиком)
2 фото: горизонталь, вертикаль
3 фото: 1 большое + 2 маленьких справа, 1 большое + 2 снизу, 3 равных по вертикали, 3 равных по горизонтали
4 фото: сетка 2×2, 1 большое + 3 маленьких, 4 в ряд, 4 в столбец
5 фото: различные комбинации
6 фото: сетка 2×3, сетка 3×2


Каждый макет описывается массивом объектов с координатами и размерами ячеек (в относительных единицах 0..1)
Активный макет подсвечен

4.3.2 Фон

Цвет: color picker (стандартный <input type="color">) + поле HEX
Прозрачность фона: чекбокс «Прозрачный фон»

Если включён → Canvas рендерится без фона (checkerboard pattern в предпросмотре, PNG при экспорте)
Если выключен → фон закрашивается выбранным цветом


Градиент (опционально, бонус): два цвета + направление (горизонталь/вертикаль/диагональ)

4.3.3 Рамка (Border/Gap)

Ширина рамки: слайдер от 0 до 50px (шаг 1), числовое поле ввода
Цвет рамки: color picker + HEX поле
Рамка — это зазор между ячейками и отступ от края Canvas до ячеек (padding)
Цвет рамки совпадает с цветом фона ИЛИ задаётся отдельно (переключатель «Использовать цвет фона»)

4.3.4 Радиус скругления

Слайдер от 0 до 100px (шаг 1) + числовое поле
Применяется к каждой ячейке (clip ячейки по border-radius)
Обновляется в реальном времени на Canvas

4.3.5 Размер холста

Два поля: Ширина × Высота в пикселях
Кнопки быстрых пресетов: 1:1 (1200×1200), 4:3 (1600×1200), 9:16 (1080×1920), 16:9 (1920×1080)
Чекбокс «Сохранять пропорции»


5. Логика макетов
Каждый макет — массив объектов типа LayoutCell:
typescriptinterface LayoutCell {
  id: number;
  x: number;       // от 0 до 1 (относительно ширины холста)
  y: number;       // от 0 до 1 (относительно высоты холста)
  width: number;   // от 0 до 1
  height: number;  // от 0 до 1
}
При рендере Canvas для каждой ячейки:

Вычислить абсолютные px-координаты с учётом padding (рамки по краям) и gap (зазор между ячейками)
ctx.save() → ctx.beginPath() → roundRect(x, y, w, h, radius) → ctx.clip()
Нарисовать фото внутри clipping region с object-fit: cover (вычислить масштаб и смещение)
ctx.restore()


6. Экспорт
Кнопка «Скачать» в шапке:

Открывает модальное окно с выбором:

PNG (с прозрачным фоном, если включён)
JPG (качество слайдер 60–100%, белый или выбранный фон)


Размер экспорта = размер Canvas (по умолчанию 1200×1200)
Опция «Увеличенный экспорт» — множитель ×2 или ×3 (экспорт в 2400×2400 при холсте 1200×1200)
Имя файла: collage_YYYYMMDD_HHmmss.png

Реализация:
javascriptconst dataUrl = canvas.toDataURL('image/png');

7. State Management
Единый стейт приложения (React Context или Zustand):
typescriptinterface AppState {
  // Холст
  canvasWidth: number;
  canvasHeight: number;

  // Макет
  selectedLayout: LayoutConfig;

  // Ячейки
  cells: {
    [cellId: number]: {
      imageId: string | null;
      offsetX: number;   // смещение фото внутри ячейки
      offsetY: number;
      scale: number;
    }
  };

  // Библиотека фото
  images: {
    [imageId: string]: {
      src: string;       // dataURL или ObjectURL
      width: number;
      height: number;
    }
  };

  // Настройки внешнего вида
  borderWidth: number;
  borderColor: string;
  backgroundColor: string;
  transparentBackground: boolean;
  borderRadius: number;
}

8. UX-детали


Адаптивность: рабочая область масштабируется под окно браузера (CSS transform scale), но внутренний Canvas — всегда в реальных пикселях
Пустые ячейки: отображаются как серый прямоугольник с иконкой «+» и текстом «Добавить фото»
Перестановка фото: drag между ячейками меняет фото местами
Загрузочный индикатор: при обработке больших файлов (>5MB)



10. Критерии готовности

 Загрузка 2+ фотографий через drag & drop и через кнопку выбора файлов
 Выбор любого из 16 макетов с моментальным применением
 Настройка радиуса скругления в реальном времени
 Настройка ширины и цвета рамки в реальном времени
 Настройка цвета фона и режима прозрачного фона
 Экспорт PNG с прозрачным фоном (без фона)
 Экспорт JPG с выбранным фоном
 Перетаскивание фото из панели в ячейки
 Перемещение фото внутри ячейки (drag)
 Работает без backend, полностью в браузере
 Производительность: рендер обновляется за < 16ms (60fps) при изменении слайдеров
